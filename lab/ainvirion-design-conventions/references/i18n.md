# Internationalization (i18n)

All AInvirion product landings must support at least English and Spanish. English is the default language.

## Requirements

- **Minimum languages**: English (`en`), Spanish (`es`)
- **Default language**: English
- **`lang` attribute**: set `<html lang="en">` (or `es`) — required for screen readers and search engines
- **`dir` attribute**: `ltr` for both English and Spanish (add `dir="rtl"` only if a RTL language is added later)

## HTML structure

Set the `lang` attribute on `<html>` and use `data-lang` for JS-driven language switching:

```html
<html lang="en" data-lang="en">
```

### Translatable content

Mark all user-facing text with a `data-i18n` attribute keyed to the translation dictionary:

```html
<h1 data-i18n="hero.title">Transform your workflow</h1>
<p data-i18n="hero.subtitle">AI-powered automation for modern teams</p>
```

The English text lives inline as the default fallback. Other languages are loaded from a translations file.

### Language switcher

Place in the navbar actions zone (`.island-actions`) or footer. Minimal UI — a `<select>` or compact toggle:

```html
<select aria-label="Language" data-lang-switcher>
  <option value="en">EN</option>
  <option value="es">ES</option>
</select>
```

Store the user's choice in `localStorage` key `lang`. On page load, check `localStorage.lang` before rendering.

## Translation files

Use JSON files with flat dot-notation keys, one per language:

```
locales/
├── en.json
└── es.json
```

Example `en.json`:

```json
{
  "nav.features": "Features",
  "nav.pricing": "Pricing",
  "nav.support": "Get support",
  "hero.title": "Transform your workflow",
  "hero.subtitle": "AI-powered automation for modern teams",
  "cta.primary": "Get started",
  "cta.secondary": "Learn more",
  "footer.copyright": "© {year} AInvirion LLC · Spokane, WA",
  "footer.privacy": "Privacy Policy",
  "footer.terms": "Terms of Service"
}
```

Example `es.json`:

```json
{
  "nav.features": "Características",
  "nav.pricing": "Precios",
  "nav.support": "Obtener soporte",
  "hero.title": "Transforma tu flujo de trabajo",
  "hero.subtitle": "Automatización con IA para equipos modernos",
  "cta.primary": "Comenzar",
  "cta.secondary": "Saber más",
  "footer.copyright": "© {year} AInvirion LLC · Spokane, WA",
  "footer.privacy": "Política de Privacidad",
  "footer.terms": "Términos de Servicio"
}
```

### Key naming convention

Use dot-notation grouped by section: `section.element`

| Prefix | Scope |
|--------|-------|
| `nav.*` | Navbar items |
| `hero.*` | Hero section |
| `features.*` | Features section |
| `cta.*` | Call-to-action buttons |
| `footer.*` | Footer content |
| `common.*` | Shared strings (e.g., "Close", "Loading") |

## JavaScript pattern

Minimal i18n function — no library needed for static landings:

```js
async function loadLang(lang) {
  const res = await fetch(`/locales/${lang}.json`);
  const dict = await res.json();
  document.querySelectorAll('[data-i18n]').forEach(el => {
    const key = el.dataset.i18n;
    if (dict[key]) el.textContent = dict[key];
  });
  document.documentElement.lang = lang;
  document.documentElement.dataset.lang = lang;
  localStorage.setItem('lang', lang);
}

// On page load
const lang = localStorage.getItem('lang') || 'en';
if (lang !== 'en') loadLang(lang);
```

## Non-translatable elements

These stay in English regardless of language:

- Company name: **AInvirion LLC**
- Copyright line (except "Privacy Policy" / "Terms of Service" labels)
- Product names (AgenticJunior, yt2txt, Cliquey, Kenos)
- Technical terms in morphs (`aria-hidden="true"` content is not translated)
- Code snippets and terminal output

## SEO considerations

- Set `<link rel="alternate" hreflang="en" href="...">` and `hreflang="es"` in `<head>`
- Use `<meta name="description">` in both languages (swap via JS or serve separate pages)
- For static sites: consider separate `/es/` path prefix if SEO is critical

## Checklist

When creating or reviewing a landing:

- [ ] `<html lang="en">` is set
- [ ] All user-facing text has `data-i18n` attribute
- [ ] `en.json` and `es.json` exist in `locales/`
- [ ] Language switcher is accessible (keyboard + screen reader)
- [ ] `localStorage` persists language choice
- [ ] Product names and company name are NOT translated
- [ ] `hreflang` alternates are in `<head>`
